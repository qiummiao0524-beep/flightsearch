# AI 航班搜索助理 - 实用功能与操作指南

## 一、工具简介
**AI 国际航班搜索助理** 是一款基于大语言模型（LLM）驱动的智能交互工具，跳过复杂的表单过滤界面，让您通过"大白话"直接和系统对话。它旨在帮助研测人员快速获取或构造特定业务数据：
- **语义化搜索**：支持极其复杂的行程组合（如日期相对词、特定航司/舱位限制）。
- **数据双模态**：系统支持“搜索查询”与“数据Mock”双模式。搜索查询支持完整的下单业务流，Mock用于极速造数测试，不受资源及网关过滤限制。
- **智能化澄清**：由于具备会话记忆，当您的需求缺失要素时，它会通过卡片追问，让补录输入像和真人聊天一样丝滑。
---

## 二、实用功能亮点

### 1. 随心所欲的“口语化”机票搜索
- **灵活的城市与机场识别**：支持输入热门城市名称（如“上海”、“香港”）或专业机场三字码（如“PVG”、“HKG”）。
- **聪明的日期推算**：除了支持具体的日期（“10月1日”），还支持对话式的相对词汇，如“明天”、“下周五”、“端午节”、“国庆”。
- **多中转联程解析**：根据底层航班数据结构，能够精准解析并展示直飞、经停或多次中转的复杂行程组合。

### 2. 精确的乘机人与舱位报价展示
当您在验证不同客群或级别报价时，系统能够智能识别并分流：
- **多客组合报价结构**：支持一句话带入乘客结构组合，如“2个大人，1个小孩”。查询后的人机界面会清晰展示总价，以及成人、儿童的单点运价差异明细。
- **自定义舱位屏蔽**：可直接在对话中指明“头等舱”、“商务舱”或“超级经济舱”。系统将专门为您过滤出对应的舱位机票结果。

### 3. 多渠道筛选与 Mock 兜底演示
- **多业务渠道定向查价**：您可以直接指定“去日本的微信 H5 渠道机票”或“去深圳的携程 APP 专机”。系统会智能锁定对应的底层售票代理渠道。
- **假数据 (Mock) 拦截保护**：当您设定的组合条件过于严酷（极冷偏航线、指定罕见渠道等），导致业务层无此真实航班下发时，系统不会出现体验不佳的错误提示或白屏。而是能自动“捏造”出一份符合逻辑的中转虚拟航班，并打上醒目的 `[MOCK]` 标签。您可以用它来验证特定 UI 卡片的渲染机制与布局逻辑。

### 4. 舒心的“防呆”多轮追问交互
- **缺失要素自动澄清**：如果您的话意图不全，比如只说“帮我查下去成都的机票”，系统不会直接中断服务，而是利用上下文记忆能力给您返回一张操作友好的 **“澄清卡片 (Clarify Card)”**，温柔地追问“请问您具体需要从哪儿出发？哪一天启程？”，点击卡片选项完成补充后，流程自动接续执行。

## 三、操作演示与典型对话示例

进入主操作页面后，您可以在聊天框中尝试以下三种经典测试场景：

### 🎯 场景 A：条件齐全，一语即达
- **👤 您的输入**：`“明天早上上海飞东京，2个大人1个小孩，要看头等舱”`
- **🤖 系统效果**：系统一步到位完成解析，直接在会话流输出对应日期上海直飞或中转东京航线的高级客舱票价列表。卡片上清晰陈列这 2 位成人和 1 位儿童的分类票价及累加明细总额。

### 🎯 场景 B：触发澄清，多轮追问
- **👤 您的输入**：`“明晚去首尔的特价票”`
- **🤖 系统效果**：系统识别出缺少核心信息（出发城市位置不明），中止航班网络请求，并为您弹出包含表单交互的澄清卡片提问：“请问您的出发城市是？”。当您在卡片里输入“杭州”后，查询正常接续并呈现杭州到首尔的航班动态。

### 🎯 场景 C：指定狭窄冷门渠道触发 Mock 演示
- **👤 您的输入**：`“下周二北京飞往悉尼的微信端小程序专属特惠航班”`
- **🤖 系统效果**：微信特惠等孤立长尾渠道在此航线上通常较难真实返回，业务进入分流兜底层降级为 Mock 模式。您将迅速看到系统生造出的包含中转站的虚假航班记录展板，右侧票价旁会闪烁鲜明的 `[MOCK]` 角标提示当前为阻断造假环境，证明渠道请求挂载测试通过。

### 🎯 场景 D：机场代码直输与具体定语约束
- **👤 您的输入**：`“PVG 到 ICN，只要韩亚航空的商务舱直飞”`
- **🤖 系统效果**：系统通过内置的 `city_mapping` 词典字典，精准将 `PVG` 转译为上海浦东机场、`ICN` 翻译为首尔仁川机场，同时挂载出“韩亚航空”、“直飞限制”与“商务客舱”的搜索参数进行底层过滤，最终展现完全吻合的高级航班列表。

### 🎯 场景 E：节假日等模糊相对时间推算
- **👤 您的输入**：`“今年国庆节从广州去曼谷，全家两大一小”`
- **🤖 系统效果**：系统中的时间智能解析引擎会自动把“国庆节”推算翻译为 `2026-10-01`，同时正确摘取“2名成人、1名儿童”并请求运价系统，展示 10 月 1 日当天的实时价格和加总明细。对于“五一”、“除夕”、“下月初”等常用口语均可顺畅识别。

### 🎯 场景 F：多轮对话记忆与状态追更 (Session Store)
- **👤 您在搜完某个航班后的输入**：`“帮我把出发时间改到下周五”` 或 `“那换成经济舱呢？”`
- **🤖 系统效果**：得益于系统的状态缓存记忆机制，它不需要您像填表单一样把“上海飞东京”、“2大1小”等历史要素重新说一遍，而是直接沿用上一轮的搜索主体，仅替换时间戳或舱位等级变量再次为您实时发起检索，实现像真人客服一般的丝滑连续体验。

## 四、测试造数进阶指南（数据工厂模式）

对于研发和测试同学，本工具更是造假数据、验证系统边界和前端排版极限的绝佳利器。您可以通过极其变态的口语组合，强制让 Mock 引擎吐出复杂的数据结构：

### 🛠️ 造数场景 1：极端多段中转排版测试
- **👤 您的输入**：`“查一下下周三 MU5001/MU5002/MU5003 从上海去巴黎的单程，经曼谷和迪拜两次中转”`
- **🤖 测试价值**：此语句会自动匹配 3 个航班号与 2 个中转城市，触发底层的级联 Mock 造数引擎。您可以利用被制造出的具有 2 个中转站、3 个独立航段的航班数据，来测试**超长航段的前端卡片伸缩、中转时长计算逻辑和多标签布局换行问题**。如果您懒得想城市，也可以直接说“转两次机，城市随便”，系统会自动塞入合理的中转三字码。

### 🛠️ 造数场景 2：极端多级客群报价边界
- **👤 您的输入**：`“明天北京去洛杉矶，9个大人，9个小孩，2个婴儿，要头等舱票”`
- **🤖 测试价值**：快速压测业务系统的乘客上限（或用来校验前端多类票卡堆叠时的金额显示是否截断）。由于系统对人群进行了细致地分发乘数算价计算（如成人数*全价 + 儿童数*7.5折系数 + 婴儿数*0.1折系数），你可以完美拿到一份超大面额、包含3种不同价格分类明细打底 JSON，校验金额千分位逗号展示和总价加权正确性。

### 🛠️ 造数场景 3：无回程组合的往返澄清流中断造数
- **👤 您的输入**：`“我要明天去伦敦的往返，回来时间还没定，必须买头等舱”`
- **🤖 测试价值**：验证业务流打断补偿与反向覆盖交互。系统在解析往返（RT）时发现回程日期 null，一定会阻断发往后端的查询请求并抛出日历卡要求您点选回程。通过这样的测试例，能迅速校验往返票业务链在异常残缺时的阻断鲁棒性。

### 🛠️ 造数场景 4：超复杂往返多中转结构打底
- **👤 您的输入**：`“下周去巴黎的往返，来回都要转两次机，城市随便”`
- **🤖 测试价值**：去程和回程各包含 3 个航段，系统将瞬间计算并吐出一组包含了 6 段子航班、且日期顺序严密的超级 Mock 结构。这对于大面积 UI 列表的渲染性能测试、往返双向中转时间计算有着极高的验证价值。

- **👤 您的输入**：`“只看明天 MU5101 这个航班的微信特惠渠道”`
- **🤖 测试价值**：精确约束底层请求参数，测试当单一结果被渲染时 UI 是否存在展示异常。同时验证非常规 `flatType` 渠道标识是否能准确从自然语言被透传到底层并回显在 Mock 标签链上。

## 三、重要备注
> [!IMPORTANT]
> - **正常查询到的数据**：直连供应商接口，反映真实运价存量，**支持完整的下单业务流**。
> - **Mock 数据**：由数据工厂模拟产出，旨在极速生成各种复杂的测试场景，其不受资源及网关层过滤限制，**暂不支持下单**。
